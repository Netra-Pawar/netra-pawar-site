# Retrieval Evaluation Metrics

## Retrieval metrics

Measure how well the embedding + retrieval pipeline surfaces relevant content:

| Metric | What it measures | When to use |
|---|---|---|
| **Recall@k** | Fraction of relevant documents in the top-k results | Primary metric — does the correct content appear in the candidate set? |
| **Precision@k** | Fraction of top-k results that are relevant | High precision = less noise for the model to filter through |
| **MRR** (Mean Reciprocal Rank) | Average position of the first correct result | Matters when the model primarily uses the top-1 or top-2 chunks |
| **nDCG** (Normalized Discounted Cumulative Gain) | Ranking quality, weighted by position | Use when the ordering within top-k matters, not just presence |
| **No-result rate** | Fraction of queries that return zero results | Indicates coverage gaps in the corpus or overly aggressive filtering |
| **Stale-result rate** | Fraction of results where the source has been updated since indexing | Tracks index freshness |
| **Retrieval latency (p50/p95/p99)** | Time from query to candidate set | Embedding inference + ANN search + filter time |

## End-to-end metrics

These measure the complete RAG pipeline, not just retrieval:

| Metric | What it measures |
|---|---|
| **Answer acceptance rate** | How often users accept the generated answer (via thumbs up, follow-up behavior, or absence of reformulation) |
| **Grounded-answer rate** | How often the model's answer is traceable to a retrieved chunk |
| **Escalation rate** | How often users escalate to human support after receiving an answer |
| **Source correctness** | How often the cited source is the canonical, current version |

## Measure retrieval before generation

If you only measure end-to-end quality, you cannot distinguish a retrieval failure (right chunk never surfaced) from a generation failure (model ignored or misinterpreted the chunk). Evaluate both stages independently before optimizing either.

The [Search Toolkit](/studio-api/knowledge-rag/search-toolkit) supports evaluation with recall, precision, MRR, and nDCG against custom test sets.
