# Set Up Your Environment

Before making your first API call to Mistral, you need to authenticate your requests and set up your local development environment. 

This guide covers the prerequisites required to interact with the `chat/completions` endpoint.

## 1. Get Your API Key

Every request to the Mistral API requires authentication via an API key.

1. Navigate to [La Plateforme](https://console.mistral.ai/).
2. Create an account or log in.
3. Go to **API Keys** in the left sidebar.
4. Click **Create new key**.
5. Copy the key immediately. For security reasons, you will not be able to view it again.

> **⚠️ Caveat: Free Tier Limits**
> New accounts start on the Free Tier. This tier is strictly rate-limited to 1 request per second (RPS) and 500,000 tokens per month. If you are evaluating Mistral for a production workload, you should upgrade to a paid tier or request workspace access to avoid `429 Too Many Requests` errors during testing.

## 2. Secure Your Key

Never hardcode your API key into your application source code, and **never commit it to version control** (like GitHub). 

The industry standard practice is to store the key as an environment variable. Open your terminal and run the following command, replacing `your_api_key_here` with the key you just generated:

### macOS / Linux
```bash
export MISTRAL_API_KEY="your_api_key_here"
```

### Windows (Command Prompt)
```cmd
set MISTRAL_API_KEY="your_api_key_here"
```

### Windows (PowerShell)
```powershell
$env:MISTRAL_API_KEY="your_api_key_here"
```

To verify the key is set correctly, print it back to the terminal:
```bash
echo $MISTRAL_API_KEY
```

## 3. Choose Your Tool

You can interact with the Mistral API using any tool that sends HTTP requests. For this guide, we will start with `curl` to understand the raw HTTP mechanics, then move to the official SDKs.

Ensure you have one of the following installed:
- **curl:** Pre-installed on macOS and Linux. For Windows, use Windows Subsystem for Linux (WSL) or download it from [curl.se](https://curl.se/).
- **Python:** Version 3.9 or higher.
- **Node.js:** Version 18 or higher (for TypeScript/JavaScript).

## 4. Understand Model Versioning

When you make a request, you must specify which Mistral model you want to use. 

Mistral uses **aliases** (like `mistral-small-latest`) that automatically point to the newest underlying model version (e.g., `mistral-small-2402`). 

> **⚠️ Caveat: Aliases vs. Pinned Versions**
> For this tutorial, we will use the `mistral-small-latest` alias. However, when you move to production, you should **pin your model version** (e.g., `mistral-small-2402`). Aliases are updated dynamically; if a new model version behaves differently, it could break your application unexpectedly.
