# Experiments

This section is where I will build small, practical experiments around documentation engineering.

The goal is not to present finished case studies too early. The goal is to explore real questions through hands-on builds, lightweight prototypes, and documented learning.

## Current themes

- How documentation can stay accurate as products change
- How API contracts, examples, and quickstarts can drift apart
- How documentation can become more useful for search, support workflows, and AI assistants
- How quality signals can help documentation teams make better decisions

## First experiment

The first experiment will look at a simple but important documentation reliability question:

> What kind of drift can an API contract catch, and what kind of drift does it miss?

This will become a small hands-on project using an OpenAPI file, a quickstart example, and a few automated checks.
